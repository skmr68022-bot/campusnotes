1. Company Setup & Configuration (Theory)

Creating a company is the mandatory first step in any computerized accounting software (like Tally, QuickBooks, or SAP). This process establishes the digital container where all financial data, masters, and transactions for a specific business entity will be stored.

📁 Directory/Path

Define where the data file (.dat or .tpy) will be saved on the hard drive.

📋 Company Profile

Input the legal name, mailing address, state, country, and contact details (PIN code, phone, email).

📅 Financial Period

Define the "Financial Year Beginning From" and "Books Beginning From" dates (essential for the software to calculate depreciation and closing entries).

🔐 Security Control: Establish a password and user management system (the 'Admin' account) to ensure data confidentiality.

Configure and Feature Settings

📊 Accounting Features

Toggle options like Multi-currency support, Cost Centers, Budgeting, and Interest Calculation based on specific debtor/creditor terms.

📦 Inventory Features

Enable features like Multiple Godowns, Batch-wise details, Re-order levels, and unit-of-measure conversions for manufacturing setups.

⚖️ Statutory Features

Configure tax parameters such as GST (Goods and Services Tax), TDS (Tax Deducted at Source), or VAT, depending on the regulatory region.

Selecting and Shutting a Company

OperationDescriptionOperational Logic
Selecting a CompanyOpening an existing database for data entry or report viewing.The software displays a list of available companies; the user selects the relevant one and authenticates with credentials.
Shutting a CompanyClosing the currently open company session.This removes the company from the active session without deleting the file. It is essential when switching between different business entities to prevent accidental data entry in the wrong company file.

2. Master Data Creation (Theory)

In computerized accounting, you don't just "create an account"; you create a Ledger and assign it to a Group. A Group acts as a primary classification, ensuring that when you create a ledger, the software automatically knows where to place it in the Final Accounts.

📒 Accounting Ledgers and Groups

  • Groups: These are the "parent" classifications (e.g., Sundry Debtors, Current Assets, Indirect Expenses, Capital Account).
  • Ledgers: These are the "child" accounts (e.g., 'Ram & Co.' under Sundry Debtors, 'Electricity Bill' under Indirect Expenses).
⚡ Crucial Logic: If you assign a ledger to the wrong group (e.g., placing 'Rent Paid' under 'Current Assets' instead of 'Indirect Expenses'), your Balance Sheet and P&L will be fundamentally incorrect.

📦 Stock Items and Groups

  • Stock Groups: Category-level classifications (e.g., "Electronics," "Stationery," "Raw Materials").
  • Stock Items: The specific, granular product (e.g., "Dell Laptop i5," "A4 Printing Paper").
  • Units of Measure: Vital configuration: define if the item is tracked in 'Nos', 'Kg', 'Meters', or 'Litres'.

3. Voucher Entry including GST (Theory)

In a Computerized Accounting System (CAS), you don't enter "Journal Entries" directly. You enter Vouchers. A voucher is a pre-formatted entry screen that asks for specific information (Date, Ledger, Amount, Narrations) and ensures the entry follows double-entry rules before saving.

Voucher TypePurpose
Contra (F4)Cash/Bank transfers (Cash to Bank, Bank to Cash).
Payment (F5)All payments made to creditors or for expenses.
Receipt (F6)All income received from debtors or through sales.
Journal (F7)Adjustments, depreciation, and credit purchases of assets.
Sales (F8) / Purchase (F9)Recording actual trading transactions.

Accounting Treatment of GST

Under GST, the tax is categorized into Input (on purchases) and Output (on sales). The accounting software automates the tax calculation, but you must ensure the correct ledger mapping for the system to handle the tax offsets correctly.

Input GST (Debit)

When you purchase goods, you pay tax. This is an asset (recoverable credit) until used to settle your own output liability.

Output GST (Credit)

When you sell goods, you collect tax. This is a liability (money owed to the government).

Offsetting

At the end of the tax period, the software performs a "Journal Voucher" to debit Output GST and credit Input GST to settle the tax liability.

⚡ Exam Pro-Tip: Always remember that in GST accounting, "Input Tax Credit" (ITC) is your primary mechanism for cost reduction. If you purchase an item for ₹100 + ₹18 GST, the Ledger cost is ₹100, and the GST is treated separately as an "Input CGST/SGST" asset ledger.

Voucher Entry Workflow

  • Selection: Choose the correct voucher type based on the nature of the transaction.
  • Date Entry: Ensure the date matches the physical invoice date (crucial for GST reporting).
  • Ledger Selection: Pick the correct Ledger (e.g., Party Name, Cash/Bank A/c).
  • GST Details: If GST is enabled, the software will pop up a window to select the correct Tax Rate (e.g., 5%, 12%, 18%) and the nature of the transaction (e.g., Intra-State vs. Inter-State).
  • Narration: Never skip the narration! It provides the legal context for future audits.
4. Generating Reports (Theory)

In a CAS, reports are dynamic and drill-down capable. You don't just see the final figure; you can click on any report item to view the underlying vouchers that generated it.

📒 Cash Book & Ledgers

Cash Book: A real-time view of all cash and bank transactions.
Ledger Accounts: The software aggregates every transaction for a specific party, giving an instant 'Current Balance'.

📊 Trial Balance

The definitive test of arithmetical accuracy. In a CAS, the Trial Balance is always "live"—as soon as you save a voucher, the Trial Balance updates automatically.

📈 Final Accounts

P&L A/c: Generates the profit/loss based on the grouping you defined in the Masters.
Balance Sheet: A snapshot of assets/liabilities, automatically adjusted for closing stock and current liabilities.

💰 Cash Flow Statement

Generated using the Indirect Method (usually). Highlights cash movements in Operating, Investing, and Financing activities.

5. Data Management & Security (Theory)

Data Backup is the process of creating a secure, secondary copy of the entire company database. In a Computerized Accounting System (CAS), all your financial records exist as digital files; if these files are corrupted or lost, the entire accounting history is gone. Backup ensures business continuity and disaster recovery.

⚡ The 3-2-1 Backup Strategy: To achieve maximum data security, follow this industry-standard rule:
  • Keep at least 3 total copies of your data.
  • Store them on 2 different types of media (e.g., local hard drive and cloud storage).
  • Keep 1 copy completely off-site (e.g., a secure remote server or encrypted cloud).

📤 Backup (Preventative)

  • Automated scheduling
  • Integrity verification
  • Encryption

📥 Restore (Corrective)

  • Locate backup file
  • Select archive
  • Target directory
  • Post-restore validation

🔐 Security Measures

  • User Role Management
  • Audit Trail (immutable log)
  • Data Encryption
Exam tip: Backup (Preventative) — an "Outgoing" process that protects against future loss. Restore (Corrective) — an "Incoming" process that fixes current damage by reverting the system to a previously known "healthy" state.

6. GST — Order of Utilization of Input Tax Credit (ITC) — Numerical Problem
Input IGSTIGST → CGST → SGST
Input CGSTCGST → IGST ❌ Cannot set off against SGST
Input SGSTSGST → IGST ❌ Cannot set off against CGST
Final RuleIGST can go anywhere; CGST & SGST can NEVER swap with each other
🚀 Exam Memory Trick: "IGST → IGST/CGST/SGST; CGST → CGST/IGST; SGST → SGST/IGST. CGST and SGST can NEVER cross-set-off with each other!"
PYQ-Style Question

M/s. Sharma Traders has the following Input Tax Credit (ITC) available in their electronic credit ledger as on 31st March 2024:

Input Tax Credit AvailableAmount (₹)
Input IGST50,000
Input CGST80,000
Input SGST60,000

The Output GST payable as per the electronic liability register is as follows:

Output GST PayableAmount (₹)
Outward IGST70,000
Outward CGST40,000
Outward SGST30,000

Required: Determine the order of utilization of ITC and prepare a statement showing the amount due, adjusted, and payable in cash.

Step‑by‑step Solution
Step 1 · Start with Input IGST (₹50,000)
Input IGST set off against Output IGST first.
Output IGST = 70,000 – 50,000 = ₹20,000 left
Input IGST = ₹0 (fully used)
Step 2 · Use Input CGST (₹80,000)
Input CGST set off against Output CGST first.
Output CGST = 40,000 – 40,000 = ₹0 paid
Input CGST left = 80,000 – 40,000 = ₹40,000
Step 3 · Use remaining Input CGST (₹40,000) against Output IGST
Balance Output IGST = 20,000 – 20,000 = ₹0 paid
Input CGST left = 40,000 – 20,000 = ₹20,000
Step 4 · Use Input SGST (₹60,000)
Input SGST set off against Output SGST first.
Output SGST = 30,000 – 30,000 = ₹0 paid
Input SGST left = 60,000 – 30,000 = ₹30,000
Step 5 · Remaining Input SGST (₹30,000)
Balance Output IGST = ₹0 (already paid). So ₹30,000 cannot be utilized and will be carried forward/lapse.
Step 6 · Prepare summary statement

📌 Quick Revision — GST Exam Cheat Sheet:
• Input IGST → 1. Output IGST → 2. Output CGST → 3. Output SGST
• Input CGST → 1. Output CGST → 2. Output IGST ❌ Cannot set off against SGST
• Input SGST → 1. Output SGST → 2. Output IGST ❌ Cannot set off against CGST
